# GazeDeck

Real-time screen gaze tracking system using Pupil Labs hardware and AprilTag markers for precise surface mapping.

## Features

- **Real-time Gaze Tracking**: Stream gaze data mapped to screen coordinates using Pupil Labs eye trackers
- **AprilTag Surface Mapping**: Generate and detect AprilTag markers for accurate surface calibration
- **Configurable AprilTag Detection**: Fine-tune detection parameters for optimal performance in various environments
- **WebSocket Streaming**: Broadcast gaze data to multiple clients in real-time
- **Multi-device Support**: Handle multiple Pupil Labs devices simultaneously
- **Flexible Surface Configuration**: Generate custom AprilTag layouts for different screen sizes
- **YAML Configuration**: Save and load surface configurations easily
- **PNG Generation**: Generate AprilTag images for printing and placement

## Installation

### Requirements

- Python 3.8+
- Pupil Labs eye tracker hardware
- **Network**: Pupil Labs eye tracker must be on the same local network as this application
- AprilTag markers (generated by this package)

## Usage

### Command Line Interface

GazeDeck provides a comprehensive CLI for all operations:

#### Generate Surface Layout

Generate AprilTag markers arranged in a grid pattern for surface mapping:

```bash
# Basic usage - generate 3x5 grid for 1920x1080 screen
python -m gazedeck generate-surface my_screen

# Custom resolution and layout
python -m gazedeck generate-surface my_screen --surface-width 2560 --surface-height 1440 --rows 4 --columns 6

# Advanced customization
python -m gazedeck generate-surface my_screen \
    --surface-width 3440 \
    --surface-height 1440 \
    --tag-size 120 \
    --rows 3 \
    --columns 8 \
    --margin 60 \
    --starting-tag-id 100

# Markers outside visible frame (negative margin)
python -m gazedeck generate-surface external_markers --margin -50
```

#### Test Device Discovery

Discover and label Pupil Labs devices connected to your network:

```bash
# Discover devices for 15 seconds
python -m gazedeck test-device-discovery --duration 15
```

#### Test Surface Layout Discovery

Discover and label surface layouts from YAML configuration files:

```bash
# Search current directory
python -m gazedeck test-surface-layout-discovery

# Search specific directory
python -m gazedeck test-surface-layout-discovery --directory ./output
```

#### Stream Gaze Data

Start real-time gaze streaming with surface mapping:

```bash
# Stream with default settings
python -m gazedeck stream
```

```bash
# Custom directory and discovery time
python -m gazedeck stream --directory ./surfaces --duration 20
```

```bash
# Fine-tune AprilTag detection performance
python -m gazedeck stream \
    --apriltag-nthreads 8 \
    --apriltag-quad-decimate 0.5 \
    --apriltag-debug 1
```

**AprilTag Detection Parameters**:
- **`--apriltag-nthreads`**: Increase for better performance on multi-core systems
- **`--apriltag-quad-decimate`**: Lower values (0.5-1.0) improve detection at the cost of speed
- **`--apriltag-decode-sharpening`**: Increase (0.1-1.0) for noisy or low-contrast markers
- **`--apriltag-quad-sigma`**: Gaussian blur for quad detection (0.0-2.0)
- **`--apriltag-debug`**: Enable debug output (0=off, 1=basic, 2=detailed)

### Surface Layout Configuration Format

The `surface_layout.yaml` file contains:

```yaml
id: my_screen
size:
  - 1920.0          # Surface width
  - 1080.0          # Surface height
tags:
  0:                # AprilTag ID
    - [-25.0, -25.0]   # Corner coordinates [x, y]
    - [75.0, -25.0]
    - [75.0, 75.0]
    - [-25.0, 75.0]
  1:                # Next tag...
    - [442.5, -25.0]
    - [542.5, -25.0]
    - [542.5, 75.0]
    - [442.5, 75.0]
  # ... more tags
```

## WebSocket API

### Connection

Connect to the WebSocket server at `ws://localhost:8765` (default port).

### Data Format

Gaze data is streamed as binary ArrayBuffer messages with this structure:

**Message format**: `[device_id:int32, surface_id:int32, x:float32, y:float32, timestamp:float64]`

- **Size**: 24 bytes per message
- **Byte order**: Little-endian
- **Invalid data**: NaN values for x/y when surface detection fails

#### Field Descriptions

- **`device_id`**: Integer device identifier
- **`surface_id`**: Integer surface identifier
- **`x`**: Normalized X coordinate (0.0 to 1.0, left to right), or NaN if invalid
- **`y`**: Normalized Y coordinate (0.0 to 1.0, top to bottom), or NaN if invalid
- **`timestamp`**: Unix timestamp in seconds (float64)

### Example Client Implementation

```javascript
const ws = new WebSocket('ws://localhost:8765');
ws.binaryType = 'arraybuffer';

ws.onmessage = (event) => {
    const buffer = event.data;
    const view = new DataView(buffer);

    // Parse binary message
    const deviceId = view.getInt32(0, true);     // device_id
    const surfaceId = view.getInt32(4, true);    // surface_id
    const x = view.getFloat32(8, true);          // x coordinate
    const y = view.getFloat32(12, true);         // y coordinate
    const timestamp = view.getFloat64(16, true); // timestamp

    // Check if data is valid
    if (!isNaN(x) && !isNaN(y)) {
        const screenX = x * screenWidth;
        const screenY = y * screenHeight;
        updateGazeIndicator(deviceId, surfaceId, screenX, screenY);
    }
};
```

## Python API

### Core Classes

#### Surface Layout Generation

```python
from gazedeck.core.surface_layout_generation import generate_surface_layout_from_rows_and_columns, save_surface_layout

# Generate a surface layout
layout = generate_surface_layout_from_rows_and_columns(
    id="my_screen",
    rows=3,
    columns=5,
    surface_size=(1920, 1080),
    tag_size=100,
    margin=25,
    starting_tag_id=0
)

# Save to disk
save_surface_layout(layout, "output")
```

#### Device Discovery and Streaming

```python
from gazedeck.cli.setup_labeled_devices import setup_labeled_devices_cli
from gazedeck.core.streaming_gaze_mapping import stream_gaze_mapped_data

# Discover and label devices
devices = await setup_labeled_devices_cli(duration=10.0)

# Stream gaze data for a device
for device_label, device in devices.items():
    queue = await stream_gaze_mapped_data(device, surface_layouts)

    # Process gaze data
    while True:
        result = await queue.get()
        print(f"Gaze on surface: {result.surface_gaze}")
```

#### WebSocket Server

```python
from gazedeck.core.websocket_server import start_ws_server, broadcast_gaze_data
from datetime import datetime

# Start server
server, broadcaster_task = await start_ws_server(host="localhost", port=8765)

# Broadcast gaze data to all connected clients
broadcast_gaze_data(
    device_id=1,
    surface_id=2,
    x=0.5,
    y=0.3,
    timestamp=datetime.now()
)
```

## Best Practices

1. **Surface Setup**:
   - Print AprilTag markers at high resolution (300+ DPI)
   - Ensure markers are flat and well-lit
   - Position markers to cover your entire area of interest

2. **Calibration**:
   - Perform calibration with the same lighting conditions as your use case
   - Ensure the eye tracker can see both eyes clearly
   - Keep the head position relatively stable during calibration

3. **Network Setup**:
   - Use a wired network connection for best performance
   - Ensure Pupil Labs devices are on the same network segment
   - Configure firewall rules to allow WebSocket connections

4. **Performance**:
   - Monitor CPU and memory usage during streaming
   - Adjust WebSocket buffer sizes based on your network capacity
   - Consider using multiple machines for high-throughput applications

## Troubleshooting

### Common Issues

#### "No devices found during discovery"
- Ensure Pupil Labs Companion app is running
- Check network connectivity between your machine and Pupil Labs devices
- Verify firewall settings allow UDP broadcast discovery
- Try increasing the discovery duration

#### "Surface layout not detected"
- Ensure AprilTag markers are clearly visible to the eye tracker's scene camera
- Check lighting conditions - markers need good contrast
- Verify marker IDs match your surface layout configuration
- Try adjusting camera exposure settings in Pupil Labs software

#### "WebSocket connection fails"
- Verify the WebSocket server is running (check for port conflicts)
- Ensure firewall allows connections on the specified port
- Try different host bindings (0.0.0.0 vs localhost)

#### "Gaze data coordinates are inverted"
- Pupil Labs uses different coordinate systems depending on camera orientation
- Check for NaN values in x/y coordinates to detect invalid data
- Adjust coordinate transformation in your client application
- Coordinates outside 0.0-1.0 range indicate gaze outside surface bounds

### Performance Tuning

- **Reduce WebSocket buffer sizes** for lower latency
- **Increase discovery timeout** if devices are slow to respond
- **Use wired networking** for consistent performance
- **Monitor system resources** during streaming sessions

## Architecture

GazeDeck consists of several key components:

- **Device Discovery**: Automatically finds Pupil Labs devices on the network
- **Surface Layout Generation**: Creates AprilTag marker configurations
- **Gaze Mapping**: Maps 3D gaze vectors to 2D surface coordinates
- **WebSocket Broadcasting**: Binary message streaming to clients
- **Calibration Integration**: Uses Pupil Labs calibration data for accuracy
